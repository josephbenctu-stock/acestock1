# 艾斯三角收斂選股器 V8（手機友善版）

這是一個 Streamlit 版台股三角收斂選股程式，可部署成手機瀏覽器 App 使用。V8 的目標是把講義規則做成「每日可用、資料可檢查、訊號可追蹤、策略可回測、紙上交易可驗證、工程可維護」的三角策略系統。

> 免責聲明：本專案僅供教學與研究，不構成任何投資建議。交易前請用券商軟體、XQ、籌碼 K 線或 TradingView 再人工確認。


## V8 新增重點（ABC 一起強化）

### A. 資料源穩定性再強化

- Yahoo Finance 仍是主要快速資料源。
- 若 Yahoo 抓不到資料、資料過舊或格式異常，會嘗試用 TWSE / TPEx 官方月資料端點補最近日 K。
- 補抓後會與本機 OHLCV 快取合併，並持續在「資料健檢」頁顯示最新資料日、資料天數與警示。
- `scheduled_scan.py` 也同步加入官方資料 fallback，可用 `OFFICIAL_FALLBACK_MONTHS` 調整補抓月數。

### B. 每日報告與推播強化

- 「策略報告」頁新增一鍵儲存報告與推播文字。
- 可手動推送 Telegram 或 Webhook。
- GitHub Actions 排程會自動輸出 `data/reports/*.md` 與 `data/reports/*.txt`。
- 可設定 `TELEGRAM_BOT_TOKEN`、`TELEGRAM_CHAT_ID` 或 `ACE_WEBHOOK_URL`。

### C. 參數優化建議

- 「歷史/訊號追蹤」頁新增 V8 參數優化建議。
- 會根據歷史訊號後續 5/10/20 日報酬，估算較適合的最低分數門檻。
- 會統計較有效的動作、型態、等級、操作模式與偏好型態。
- 可把建議儲存到 `data/settings.json`，也可下載 JSON。
- 樣本不足時會提示只供觀察，避免過度擬合。

## V7 新增重點

### 1. 模組化工程骨架

新增 `ace_engine/` 套件，將核心邏輯拆成可測試模組：

```text
ace_engine/
  data_provider.py     # 代號清理、OHLCV清理、CSV資料讀取
  indicators.py        # MA、KD、布林、成交量指標
  triangle_engine.py   # 輕量三角偵測與三角品質分級
  signal_engine.py     # 訊號階段判斷
  risk_engine.py       # 風險報酬比與風險股數
  backtest_engine.py   # 回測摘要
  database.py          # SQLite工具
  report_engine.py     # Markdown報告
  paper_trading.py     # 紙上交易CSV工具
```

Streamlit 仍由 `app.py` 提供完整手機介面；模組化檔案讓未來要做 CLI、排程、測試或真正 App 時，不需要重寫核心。

### 2. 三角品質分級

V7 不只顯示總分，也新增：

- 型態品質：優秀 / 良好 / 普通 / 偏弱。
- 型態品質分。
- 上線觸碰、下線觸碰、線內比例、收斂幅度、成熟度與量縮品質。
- 若型態分數偏低，會降分並要求人工看圖確認。

### 3. 左側 / 右側 / 保守 / 短波段 / 中波段模式

側邊欄新增「V7 操作風格」。

- 左側模式：偏重下線不破、尾端小 K 量縮、回測 60MA 不破、KD 低檔黃金交叉、停損距離近。
- 右側模式：偏重初站上上線、突破後回測上線不破、收斂尾端、上升或對稱三角。
- 保守模式：偏重風險報酬比與成熟度，風報比不足會降分。
- 短波段模式：偏重已表態的突破與回測訊號。
- 中波段模式：偏重上升/對稱三角與可追蹤型態。

### 4. 風險報酬比與左右側分數

每檔候選股新增：

- 風報比。
- 風報判斷。
- 風險%。
- 潛在報酬%。
- 偏好型態：左側試單 / 右側突破 / 平衡觀察。
- 左側分、右側分。

這能避免「看起來有三角，但停損很遠、上方空間不夠」的訊號。

### 5. 紙上交易模式

新增「🧾 紙上交易」頁：

- 可從最近掃描結果建立模擬單。
- 記錄進場價、股數、單位數、停損價、進場訊號與備註。
- 可更新目前價格、目前訊號、目前分數、風報比、損益%與損益金額。
- 可手動結案並記錄出場價、出場備註與已實現損益。
- 資料儲存在 `data/paper_trades.csv`。

這適合先用 1–3 個月驗證訊號品質與自己的執行紀律。

### 6. 策略報告產生器

新增「📑 策略報告」頁：

- 自動產生 Markdown 盤後報告。
- 包含大盤狀態、候選股摘要、族群強度、優先候選與明日行動清單。
- 可下載 `.md` 檔，也可儲存到 `data/reports/`。

### 7. 單元測試與專案驗證

新增：

```text
scripts/validate_project.py
scripts/run_tests.py
tests/test_engine.py
```

執行：

```bash
python scripts/validate_project.py
python scripts/run_tests.py
```

可檢查核心檔案是否存在、Python 是否能編譯、模組的指標/三角/風控/訊號/紙上交易邏輯是否正常。

## V6 保留功能

### 多層資料穩定性

- Yahoo Finance 抓日 K。
- 每檔股票自動建立本機 OHLCV 快取：`data/ohlcv_cache/`。
- 資料源暫時失敗時，會嘗試讀取本機快取。
- 每檔候選股會顯示：資料狀態、最新資料日、資料天數、資料警示。
- 「資料健檢」頁可檢查單檔資料品質、快取檔、SQLite 歷史與錯誤紀錄。

### SQLite 歷史資料庫

除了 CSV 歷史外，也會同步寫入：

```text
data/ace_triangle.db
```

資料庫包含每次掃描批次、候選股、評分、等級、動作、型態、上線、下線、標籤、警示與原始 JSON。

### 不要交易濾網

會提醒：

- 大盤偏空且個股尚未突破。
- 三角開口太大。
- 風險報酬比不足。
- 股價明顯低於 60MA。
- 短線漲幅與量能過熱。

### 完整分批策略回測

近似講義流程：

- 基本單 / 試單進場。
- 突破上線加碼。
- 回測上線不破加碼。
- 跌破下線、跌破上線、獲利出場警示時出場。
- 加入手續費、證交稅、滑價。
- 顯示勝率、平均報酬、總損益、最大連虧、平均持有天數。

## 原有核心功能仍保留

### 三角收斂選股

- 自動抓台股日 K，支援手動輸入與全市場掃描。
- 偵測對稱三角、上升三角、下降三角、疑似收斂。
- 估算三角上線、下線、觸線次數、收斂成熟度。
- 顯示關鍵價位：上線、下線、60MA、布林通道、突破長紅 K 一半價。

### 講義進出場規則量化

- 收斂尾端小 K 量縮。
- 下線不破。
- 初站上上線。
- 突破後回測上線不破。
- 未突破跌破下線停損。
- 已突破跌破上線停損。
- 離開布林上軌獲利觀察。
- 翻黑長上影放量獲利觀察。
- 跌破突破長紅 K 一半價警示。

### 提高勝率條件

- 5/10/20 均線糾結。
- 布林收縮。
- 回測 60MA 不破。
- KD 低檔黃金交叉。
- 買賣家數差 CSV 匯入。
- 小資本股 20 日均量上限。
- 三角再三角模型。
- 大盤環境濾網。
- 族群強度排行。

### 掃描歷史與訊號追蹤

- 每次掃描自動儲存到 `data/scan_history.csv` 與 SQLite。
- 可匯入、匯出歷史掃描 CSV。
- 可追蹤訊號後續 1 / 3 / 5 / 10 / 20 / 60 日報酬。
- 可依「動作、型態、等級」統計勝率與平均報酬。

### 交易計畫產生器

選定候選股後，自動產生：

- 觀察價位。
- 基本單 / 試單條件。
- 突破加碼條件。
- 回測加碼條件。
- 停損參考。
- 獲利出場觀察。
- 基本單金額、股數、風險股數上限。

## 本機執行

```bash
pip install -r requirements.txt
streamlit run app.py
```

## 手機像 App 一樣使用

1. 建立 GitHub repository，放入整個資料夾內容。
2. 到 Streamlit Community Cloud 建立新 app。
3. Main file 填 `app.py`。
4. 部署完成後，用手機瀏覽器開啟網址。
5. iPhone：Safari → 分享 → 加入主畫面。Android：Chrome → 選單 → 加到主畫面。

## 買賣家數差 CSV 格式

CSV 至少需要 `code` 與 `diff` 兩欄，可選 `date`：

```csv
code,date,diff
2330,2026-06-08,-120
2330,2026-06-09,-95
```

中文欄位也可以：`股票代號, 日期, 買賣家數差`。

若資料來自 XQ 電腦版，請在側邊欄勾選「我的買賣家數差來自 XQ 電腦版」，因為講義提到 XQ 的方向相反。

## 持股 CSV 格式

```csv
code,buy_price,shares,entry_reason
2330,600,1000,三角突破
6488.TWO,50,2000,下線不破試單
```

## GitHub Actions 每日排程

1. 把 `scripts/scheduled_scan.py` 放到 repository。
2. 把 `.github/workflows/ace_daily_scan.yml` 放到 repository。
3. GitHub → Settings → Secrets and variables → Actions。
4. Variables 設定：
   - `WATCHLIST`：例如 `2330.TW,2317.TW,2454.TW`
   - `MIN_SCORE`：例如 `60`
   - `PERIOD`：例如 `18mo` 或 `5y`
5. Secrets 設定：
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_CHAT_ID`

預設排程為台灣時間約 14:40，適合收盤後掃描。

## 重要限制

- Yahoo Finance 的日 K 可能延遲、缺漏，且不等於即時盤中資料。
- 本機快取能降低資料源失敗風險，但不能保證資料一定正確。
- 買賣家數差不是免費公開穩定欄位，需要自行從 XQ、籌碼 K 線、三竹等工具匯出。
- 三角畫線有主觀性，程式只能做候選股初篩，正式進出仍要人工確認。
- 歷史追蹤與回測只能驗證過去，不代表未來績效。
